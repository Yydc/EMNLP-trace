## Key invariants

1. **Every paper number flows through `out/analysis/numbers.json`.**
   main.tex references e.g. `\OutsideGRegRCorr` (not `0.62`). The macro
   is defined in `numbers.tex` which is regenerated by `make tex` from
   `numbers.json`. Hand-typed numerics inside main.tex are CAUGHT by
   `make check`.

2. **Hard rows are derived from Full runs**, never separately run.
   Hard ⊂ Full by `problem_id`; the slicer is in
   `src/tracebench/evaluation/subset_slice.py`.

3. **Statistical reporting is task-bootstrap**, not seed-bootstrap.
   We do not run multi-seed sampling; CIs reflect uncertainty over the
   problem distribution given fixed single-run trajectories.

4. **`make check` is a hard gate.** Before `make paper` you must pass:
   - main.tex \\input's numbers.tex
   - every macro used in main.tex is defined in numbers.tex
   - no `\todo` / `TBD` / `mock` strings remain
   - main_gap_table.csv has exactly the expected row count
   - Hard ⊂ Full re-verified
