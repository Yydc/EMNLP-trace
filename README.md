# TraceBench-Full

> **TraceBench-Full: A Causally Grounded Dataset for Evaluating Traceable Multi-Turn Code Debugging.**
> ACL 2026 submission.

A passing patch is not necessarily an auditable repair. TraceBench-Full makes
repair traceability measurable by turning each multi-turn debugging trajectory
into a causally auditable object.

## Quickstart (server-side)

```bash
# 0. Clone + install
git clone <repo> tracebench && cd tracebench
make install        # pip install -r code/requirements-eval.txt
# for local-model evaluation also:  make install-build

# 1. Pre-flight: verify Hard ⊂ Full, Python ≥ 3.9, data files present
make verify-env

# 2. Cost estimate (no calls made; reads pipeline.yaml)
make estimate

# 3. Run the full pipeline (~4 days, ~$130 with buffer)
export GOOGLE_API_KEY=...           # required only if `gemini_31_pro` in matrix
make all

# … or stage-by-stage:
make label         # ~45 min CPU (counterfactual replay labeling)
make dry-run       # ~1 h, ~$5 (30-task Day-0 calibration)
make eval          # ~12 h H100 + ~4 h API parallel  (resumable)
make analyze       # ~30 min CPU (all CSVs + numbers.json)
make figures       # ~5 min     (3 multi-panel PDFs)
make tex           # ~1 min     (numbers.tex + tables/*.tex)
make check         # hard-fail guardrail
make paper         # pdflatex → code/paper/main.pdf
```

## The `pipeline.yaml` contract

Every script reads `pipeline.yaml`. It declares for every stage:
- **inputs**: paths that must exist
- **outputs**: paths the stage produces (the ONLY place those artifacts live)
- **commands**: shell to invoke
- **estimated**: wall-clock + cost
- **gate**: human-readable block conditions

This means anyone (you, a reviewer, a CI) can re-run any sub-graph with
a one-liner. Nothing is wired to "wherever the developer happened to put it."
