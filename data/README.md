# `data/` — TraceBench dataset (input + derived)

## What's here at clone time

| File | Size | Contents |
|------|-----:|----------|
| `tracebench_full.json` | 49 MB | 818 problems, multi-turn, with `original_code` + per-turn `target_code` + `injections` |
| `tracebench_hard.json` | 21 MB | 128 strict subset of Full; same schema. 128/128 `problem_id` overlap. |
| `oracle_spans.json`    | 46 KB | per-task `{problem_id: {file_path: [line_numbers]}}` for Hard split only |
| `splits/manifest_full.json` | 282 KB | lightweight per-problem metadata (rating / depth / tags / num_turns) |
| `splits/manifest_hard.json` |  46 KB | same for Hard |
| `reference_trajectories/qwen3_32b_trace/` | 6 MB | 128 dev-reference traces from Qwen3-32B (CLI format) |

## What gets generated by the pipeline

| File | Stage | Notes |
|------|-------|-------|
| `derived/tracebench_full_labeled.json` | `make label` | + `active_faults_per_turn` field |
| `derived/tracebench_hard_labeled.json` | `make label` | same |

`data/derived/` is gitignored.

## Entry schema (top-level keys)

```jsonc
{
  "trace_id":              "TB_00086"          ,  // Full id (Hard is TB_HARD_*)
  "problem_id":            "1054H"             ,  // PRIMARY KEY across splits
  "source":                "CodeFlowBench-2505",
  "rating":                3500                 ,  // Codeforces rating
  "difficulty":            "extreme"            ,  // {easy, medium, hard, very_hard, extreme, unrated}
  "depth":                 4                    ,  // call-graph depth
  "codeforces_tags":       ["dp","graphs"]      ,
  "multi_turn":            true                 ,  // always true
  "original_code":         "def solve(...):..." ,  // verified reference P*
  "conversation_history":  [ /* per-turn */ ]   ,
  "injections":            [ /* per-injection */ ],
  "active_faults_per_turn": null                  // ★ added by labeler
}
```

Per-turn entry:
```jsonc
{
  "turn_id":             0                       ,
  "context":             "...prior turn code..." ,
  "target_code":         "...this turn body..."  ,  // may contain injection
  "original_target_code": "...clean version..."  ,  // for counterfactual revert
  "test_cases":          [ "assert ..." ]        ,
  "has_error":           true                    ,
  "injections":          [ /* if any */ ]
}
```

## Hard ⊂ Full guarantee

Verified 2026-05-15: 128/128 Hard `problem_id`s appear in Full. Shared
problems have identical injections, turns, and target_code. The only
difference is `trace_id`: Hard renames it (`TB_HARD_00000` ↔ `TB_00086`).
This means **every model is run once on Full**; Hard rows are derived
via `code/src/evaluation/subset_slice.py:slice_records_by_problem_ids`.

To re-verify: `make verify-env`.
